# Contributing

Contributions are welcome.

The goal of this repository is clarity and educational value.

## Before Contributing

Read:

- COMMENT_STANDARDS.md
- NAMING_STANDARDS.md
- HDL_STYLE_GUIDE.md

These documents define the expected coding and documentation style.

## Contribution Guidelines

Changes should prioritize:

- clarity
- correctness
- maintainability
- educational readability

Avoid unnecessary complexity.

## Acceptable Contributions

Examples include:

- additional adder architectures
- improved testbenches
- waveform documentation
- teaching diagrams
- architecture explanations

## Pull Request Requirements

Each pull request must include:

- a clear description of the change
- updated documentation when necessary
- matching testbench updates if hardware modules change

## Code Review Philosophy

Review focuses on:

- correctness
- clarity
- alignment with project standards
